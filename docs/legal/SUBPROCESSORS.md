# QQueue Cloud Subprocessors

Effective Date: Draft - not yet effective

QQueue Cloud uses third-party subprocessors to provide, secure, and operate the
service. A subprocessor is a third party that processes Customer Data on QQueue's
behalf.

This page lists the categories of subprocessors QQueue may engage. Specific
vendors will be named here before commercial launch and kept current thereafter.

## Categories of Subprocessors

| Category | Purpose | Example data processed |
| --- | --- | --- |
| Hosting and infrastructure | Compute, storage, networking for the platform | All Customer Data |
| Database and managed data services | Persistent storage of application data | Contacts, templates, campaigns, metadata |
| Email / SMTP infrastructure | Outbound email delivery and relay | Recipient addresses, message content, delivery metadata |
| Payment processing | Billing and subscription management | Account and billing information |
| Analytics | Product usage and performance analytics | Usage and technical data |
| Error monitoring | Diagnostics and reliability | Logs, error events, technical data |
| Customer support | Handling support communications | Support content, account information |
| Security and abuse prevention | Fraud, spam, and abuse mitigation | Technical data, message metadata |

## Notice of Changes

QQueue will update this list when subprocessors are added or removed and will
provide reasonable notice of material changes so customers may object on
reasonable grounds, as described in the Data Processing Agreement
(`docs/legal/DATA_PROCESSING_AGREEMENT.md`).
